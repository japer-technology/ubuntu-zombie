# Files not required

An inventory of files that can be removed **without impacting the
product's functionality or its user-facing documentation**. These are
mostly the working documents produced *during construction* —
research, analysis, planning, brainstorming, and marketing scaffolding
— not the code, the installer, or the docs a user reads to run and
maintain Ubuntu Zombie.

Nothing here is referenced by the runtime (`scripts/`, `payload/`,
`tests/`, `Makefile`, `debian/`). Where a file *is* linked from
something that ships, it is called out under **Caveats** so the link
can be fixed in the same change.

## How this was determined

- **Functionality** = everything the installer and agent need:
  `scripts/`, `payload/`, `tests/`, `Makefile`, `VERSION`, `debian/`.
  None of the files below are part of those.
- **Shipped documentation** = the files listed in the README doc table
  and everything `make package` bundles into the release tarball
  (`scripts payload tests Makefile VERSION README.md CHANGELOG.md
  CONTRIBUTING.md CODE_OF_CONDUCT.md LICENSE .editorconfig SECURITY.md
  docs debian`). Files below that live *outside* the tarball are safe;
  files that live under `docs/` currently ship but are internal notes,
  not user documentation.
- References were checked with a repo-wide search; only `CHANGELOG.md`
  (historical mentions) and the noted caveats link to these files.

## 1. Construction / analysis notes — safe to remove

Internal essays written for coding agents, not for operators. Only
referenced from `CHANGELOG.md` history.

- `docs/analysis/improvements-1.md`
- `docs/analysis/install-enhancement-1.md`
- `docs/analysis/ubuntu-zombie-zero.md`
- `docs/RELEASE-PLAN-123.md` — a one-off "next 3 steps" plan derived
  from `docs/research/RELEASE-FUTURE.md`; not referenced anywhere.
- `docs/OPENSSF-SCORECARD.md` — a speculative "how we could score
  higher" note; not referenced anywhere.

## 2. Research background — safe to remove (see caveat)

`docs/research/` (17 files): background notes on the alternatives that
were evaluated before building Ubuntu Zombie (`ALTERNATIVE-*.md`,
`ALTERNATIVES*.md`, `SIMILAR.md`, `REQUIREMENTS.md`, `INSTALL-UX.md`,
`HERMES-CHAT-COMMANDS*.md`, `DEEPSEEK-ADVICE.md`, `ARCHITECTURE-IDEA.md`,
`RELEASE-FUTURE.md`, `UPGRADES-R1.md`, `README.md`).

**Caveat:** the README doc table links to `docs/research/`. If this
directory is removed, delete that one row in `README.md` too so no
broken link remains.

## 3. Optional-feature planning — safe to remove

`options/` (12 files): brainstorming and design sketches for features
that are **not implemented** (backup, DNS, Forgejo, inventory, local
LLM, Nextcloud, observability, proxy, remote, snapshots, plus
`brainstorm.md`). These are planning artifacts, not shipped behaviour,
and are **not** included in `make package`.

## 4. Promotion / marketing kit — safe to remove

`promotion/` (22 files): brand guidelines, launch copy, press kit,
social posts, landing-page mock-up, etc. Marketing scaffolding with no
bearing on functionality or product docs, and **not** included in
`make package`.
